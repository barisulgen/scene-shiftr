### What's new in v0.4.0

**Workspace Export & Import**

- Export any workspace as a JSON file — saves all apps, folders, URLs, system settings, sounds, and color to a single file
- Import workspaces from JSON files — creates a new workspace with a fresh ID. If a workspace with the same name already exists, "(imported)" is appended
- Exported files include a version field (`exportVersion: 1.0`) for future format compatibility
- Import validates the file structure and shows an error if the JSON is malformed or missing required fields

**Workspace Duplication**

- Duplicate any workspace with one click — creates an exact copy named "[original] (copy)"
- The duplicate gets a new unique ID and is not set as active
- Default workspace cannot be duplicated

**Workspace Color Accents**

- Each workspace gets a color from 8 preset swatches: red, orange, yellow, green, blue, purple, pink, cyan
- New workspaces are randomly assigned a color. Default workspace always uses coral
- The color shows up in four places: sidebar left border, emoji container tint, detail view header border, and activate button gradient
- Color swatches are shown inline with the name input in the create/edit form

**Screensaver Field**

- New screensaver field in workspace settings, shown side-by-side with wallpaper in the create/edit form
- Browse for .scr files or any file type
- Data model is ready — activation logic will be added in a future version

**Window Position Memory**

- The app remembers where you placed the window and restores it on next launch
- If the saved position is off-screen (monitor was disconnected), it falls back to centered on the primary display

**Feedback Popup**

- The sidebar Feedback button now opens a popup with: bug report/feature request link (GitHub issues), GitHub repository link, developer contact email, and a thank-you message

**Form Layout Improvements**

- Folders and URLs sections are now side by side
- Wallpaper and Screensaver are side by side
- Close-other-folders toggle moved into the Folders header row
- Transition Sound label added above the picker
- Form fields reordered: Audio → Transition Sound → Playlist URI → Wallpaper + Screensaver

**Bug Fixes**

- Fixed duplicate audio-controller imports in main.ts
- Fixed try block indentation in workspace-manager.ts

### Removed in v0.4.0

Nothing removed in this release.

### What's NOT included in v0.4.0

- Night light control (no safe Windows API exists)
- Monitor layout switching
- Bundled AudioDeviceCmdlets DLL (still requires one-time internet download)
- Screensaver activation (field exists in UI but not wired to system switching yet)
- Global hotkeys
- Auto-switch on device connect or time schedule

### Known issues

- Audio device switching requires first-launch internet connection to install AudioDeviceCmdlets PowerShell module (~2 seconds, one-time)
- Native select dropdown styling may vary across Windows versions

### Upgrade from v0.3.0

Delete the contents of `%APPDATA%/scene-shiftr/workspaces/` before launching v0.4.0. The Default workspace will be recreated automatically with the new `color` and `screensaver` fields.
